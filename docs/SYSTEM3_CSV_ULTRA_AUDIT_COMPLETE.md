**Frequency**: 6-9 occurrences

- Different schema than main header (missing columns)
- Causes column misalignment

**Evidence**:
```

Line 32: underlying,index_exch,opt_exch,spot,expiry,,side,symbol,token,,time_to_expiry,...

```

**Impact**:
- CSV parsing treats headers as data rows
- Column misalignment (data read from wrong columns)
- `final_score` may be read incorrectly
- Forward returns misaligned
- Phase 223 errors (partially mitigated with numeric conversion)

**Root Cause**:
- Source file (`dhan_index_ai_signals_curated.csv`) likely has duplicate headers
- Phase 221 reads curated CSV and includes headers as data
- Phase 221 writes all rows (including header rows) to output

**Fix Required**:
1. Clean CSV file (remove duplicate headers)
2. Fix Phase 221 to skip header rows when reading
3. Fix source file generation to prevent duplicate headers
