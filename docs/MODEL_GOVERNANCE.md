# Model Governance — Genesis System3

**Purpose:** Track model versions, data provenance, and rollback for ML/trading components.

---

## Model Artifact Locations

| Path | Purpose |
|------|---------|
| `outputs/` | Runtime outputs: health.json, positions_live.json, signals, QC reports |
| `outputs/models/` | Trained model artifacts (if used) |
| `config/` | Thresholds, regime config, live trade config |
| `phases/` | Phase modules (system3_phase*.py) — logic, not serialized models |

---

## Versioning

- **Phase logic:** Tracked in Git; version = commit hash or tag.
- **Model weights:** If stored, use naming: `{model_name}_v{version}_{timestamp}.{ext}`.
- **Config:** `config/` files in Git; changes tracked via commits.

---

## Data Provenance

- **Inputs:** CSV/Parquet sources in `outputs/` or configured paths.
- **Signals:** Generated by `system3_signal_engine.py`; logged to `outputs/`.
- **Positions:** `positions_live.json` + broker reconciliation.

---

## Rollback

- **Config:** `git checkout` of config files; restart backend.
- **Phase logic:** `git revert` or `git checkout`; re-run QA.
- **Model artifacts:** Restore from backup (see `docs/BACKUP_RESTORE.md`); point config to previous artifact path.

---

## Security

- **Do not load untrusted pickle/model files.** Only load models from trusted, version-controlled paths.
- **Keras/TensorFlow:** Use `safe_mode=True` when loading `.keras` from external sources (mitigation is partial; see P1_SECURITY_REMEDIATION).

---

## Checklist for New Models

- [ ] Document model path and version in config.
- [ ] Add to backup/restore procedure.
- [ ] Test rollback to previous version before deploy.
